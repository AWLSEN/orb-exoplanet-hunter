orb-exoplanet-hunter — build log
=================================

Niche locked: M-dwarf habitable-zone candidate surfacer. SPOC/QLP focus on
brighter targets + shorter periods; faint-M-dwarf FFIs are under-searched
and the HZ regime has the strongest "potentially habitable" story.

Stack decisions:
  - Python 3.10+ (every exoplanet tool — lightkurve, wotan, TLS, TRICERATOPS
    — is Python-native; rewriting in TS would be painful and add no value).
  - lang=python in orb.toml (native Orb runtime — no bin/start wrapper).
  - Z.AI GLM Coding Plan for LLM calls (ANTHROPIC_AUTH_TOKEN + BASE_URL to
    the Anthropic-compatible proxy). Cheap enough for weekly red-team.
  - HTTP_PORT (not ORB_PORT — Orb reserves that).
  - Build must `git fetch + reset` first; Orb doesn't re-clone on rebuild.
  - Resource floor: 8GB RAM / 30GB disk. Exoplanet stack is heavy + TESS
    light curves accumulate on disk.

Verification architecture (the part that makes the showcase credible):
  - 10 per-candidate hard gates; 1 fail = candidate dropped.
  - 6 pipeline-health checks nightly (known-planet recovery, injection-
    recovery sweep, null-data FP rate, ephemeris consistency, depth
    distribution sanity, Claude red-team review). Any hard check fails
    -> pipeline into read-only, operator alert.

  Batch 3 — TRICERATOPS FPP + ranking + multi-sector recurrence

Deploy gotchas accumulated this batch:

    by default or hit corrupt FITS reads on T16/QLP entries. Added per-entry
    try/except so one bad product can't poison the batch.
  - pytest integration marker needs an explicit default exclude
    (addopts = "-m 'not integration'") or the default run hits MAST.
  - cache_dir must accept str | Path at API boundary (caller ergonomics).
