An AI searching for new worlds in TESS data.

An autonomous agent runs on Orb Cloud, fetches light curves from NASA's TESS satellite, looks for the faint periodic dip a planet makes when it crosses its star, and publishes vetted candidates here. Open source. Checks its own work. Only billed while it's running.

What it's looking for

A 0.1% dip, repeating on a clock.

When a planet passes in front of its star, starlight dims by a fraction of a percent for a few hours. The dip repeats every orbital period. The agent searches millions of TESS light curves for that exact signature.

depth ≈ 200 ppm (two parts in ten thousand)
duration ≈ a few hours
period constant — every few days to weeks

Cost model

Billed for working, not waiting.

Each square is a 10-minute slice of the last 4 hours. Green = the agent was awake, processing. Gray = checkpointed to NVMe, compute cost effectively zero. A week of hunting looks like a mostly-gray grid punctuated by bursts of green.

A VPS bills the same whether the agent is busy or idle. A serverless function can't hold state between invocations. Orb Cloud freezes the entire process to disk when there's nothing to do and wakes it in seconds when there is. This page exists to make that concrete. — why we built on Orb
What it found

Candidates that cleared every gate.

Every surviving candidate ran a chain of independent checks: odd/even transit depth agreement, secondary-eclipse ruling-out, cross-match against known TESS Objects of Interest, Gaia DR3 RUWE to rule out unresolved binaries. A failure at any hard gate would drop the candidate before it ever reached this page.

Does the agent itself still work?

Pipeline health checks, last run.

Before trusting any new candidate, the agent re-runs a canonical set of known-planet recoveries, an injection-recovery sweep, and a null-data false-positive probe. If any of these breaks, the agent writes a HALT flag and refuses to publish until a human acknowledges.
